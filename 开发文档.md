# 开发文档 (Development Documentation)

## 1. 项目简介
本项目为校园失物招领系统，旨在方便师生发布和查找失物/招领信息。采用前后端分离架构。

## 2. 快速开始

### 2.1 环境准备
*   JDK 17+
*   Node.js 16+
*   MySQL 8.0
*   Redis (可选，推荐)

### 2.2 后端启动
1.  修改 `src/main/resources/application.yml` 中的数据库配置。
2.  运行 `LostAndFoundApplication.java`。
3.  Swagger 文档地址: `http://localhost:8080/doc.html` (待确认)。

### 2.3 前端启动
1.  进入 `multi-window-vue - 2.1` 目录。
2.  安装依赖: `npm install`。
3.  启动服务: `npm run dev`。

### 2.4 Docker Compose 一键启动（13.3）
1. 在项目根目录执行：`docker compose up -d --build`  
2. 启动后访问：
   * 前端：`http://localhost:5173`
   * 后端：`http://localhost:8080`
   * MySQL：`localhost:3306`
   * Redis：`localhost:6379`
3. 关闭环境：`docker compose down`
4. 清理并删除数据卷：`docker compose down -v`

## 3. 目录结构说明

### 后端 (Spring Boot)

## 5. 2026-02-25 一致性规则

### 5.1 账号与身份模型
* `accountName`：登录账号，注册后固定，不允许普通用户在 `/user/update` 中修改。
* `username`：展示昵称，允许普通用户在 `/user/update` 中修改（需唯一）。
* 个人资料页字段可编辑性：
  * 可编辑：`username`、`phone`、`email`
  * 不可编辑：`accountName`、`name`、`college`

### 5.2 密码策略
* 注册、改密统一规则：`8-20` 位，且必须同时包含字母与数字。
* 后端强校验入口：
  * `POST /auth/register`
  * `POST /auth/resetPassword`
* 密码存储：BCrypt；历史 MD5 密码在成功登录时自动升级为 BCrypt。

### 5.3 删除策略
* 以下表统一逻辑删除：`user`、`lost_info`、`find_info`、`activities`、`comment`、`comment_like`、`honor_period`、`honor_period_item`、`sys_dict_type`、`sys_dict_data`。
* MyBatis-Plus 已配置逻辑删除（`isDeleted`, `0/1`），默认查询会过滤已删除数据。
* `operation_log` 保持物理日志策略（当前不提供删除接口）。
* 管理端恢复接口（新增）：
  * `PUT /admin/comments/{id}/recover`
  * `PUT /admin/comments/batch/recover`
  * `PUT /admin/activities/{id}/recover`
  * `PUT /admin/activities/batch/recover`
  * `PUT /admin/honor/items/{id}/recover`
  * `PUT /admin/honor/periods/{periodId}/recover`

### 5.4 上传策略
* 生产环境禁止 mock 上传回退。
* 开发环境仅在显式设置 `VITE_ENABLE_MOCK_OSS=true` 时允许 mock OSS 地址。
