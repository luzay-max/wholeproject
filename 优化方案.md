# 项目优化方案 (Optimization Plan)

## 1. 概述
本方案旨在根据 `需求.txt` 对现有“失物招领系统”进行全方位优化，涵盖用户认证、表单规范、界面交互、权限管理及数据治理等核心模块。

## 2. 核心功能优化

### 2.1 用户注册与登录
*   **验证码机制**: 
    *   保留现有验证码功能。
*   **记住我 (Remember Me)**:
    *   **后端方案**: 登录成功后生成一个长期有效的 `Refresh Token` 存储于 Redis，有效期设为 7 天 (可配置)。
    *   **前端方案**: 将 Token 存储于 LocalStorage/Cookies。
    *   **交互**: 登录页增加“记住我”勾选框，勾选后请求后端颁发长期 Token。
*   **账号体系改造**:
    *   **新增字段**: `account_name` (账户名，唯一，不可改) 和 `username` (用户名/昵称，可改)。
    *   **逻辑**: 登录使用 `account_name`，页面展示使用 `username`。
    *   **老数据迁移**: 将现有 `username` 迁移至 `account_name`，并复制一份给 `username`。
    *   **新增字段**: `student_id` (学号，必填，需唯一)，用于白名单校验。
*   **注册白名单**:
    *   **Excel导入**: 管理员上传学生名单 Excel。
    *   **校验逻辑**: 注册提交的 `student_id` 和 `name` 必须存在于白名单表中，否则拒绝注册。

### 2.2 表单与输入校验
*   **字段校验规则**:
    *   `phone`: 正则校验手机号格式。
    *   `email`: 正则校验邮箱格式。
    *   `name`: 允许生僻字，禁止特殊符号 (如emoji, 乱码)，要是真实姓名。
    *   `student_id`: 必填，数字/字母组合。
    *   `password`: 字母+数字组合，长度 8-20 位。
*   **交互体验**:
    *   实现“回车切换焦点”指令 (Vue Directive)。
    *   实现“失焦查重”功能 (用户名/学号等唯一字段)。
*   **新增字段需求**:
    *   **全表通用**: `is_deleted` (逻辑删除, TinyInt, 0未删 1已删)。
    *   **失物表 (lost_info)**: 
        *   `lost_time` (丢失时间) - 校验不可晚于当前时间。
        *   `contact_info` (联系人)。
    *   **招领表 (find_info)**:
        *   `found_time` (捡到时间) - 校验不可晚于当前时间。
        *   `contact_info` (联系人)。
    *   **用户表 (user)**:
        *   `college` (学院名)。
        *   `status` (用户状态: 0正常, 1封禁)。

### 2.3 界面与交互 (UI/UX)
*   **布局优化**: 
    *   Label 前置，输入框对齐，间距紧凑化。
*   **导航优化**:
    *   管理员后台引入“多标签页” (Tabs) 模式，支持同时打开多个管理页面。
    *   公共页面增加面包屑导航 (Breadcrumb)。
*   **视觉升级**:
    *   统一色调与 UI 规范 (参考 Ant Design 或 Element Plus 标准主题)。

### 2.4 权限与内容管理
*   **发布审核流程 (重点优化)**:
    *   **发布即待审**: 普通用户发布信息后，默认状态为 `PENDING` (待审核)，仅发布者自己和管理员可见，前台列表不可见。
    *   **审核通过**: 管理员审核通过后，状态变更为 `APPROVED` (已发布)，前台列表可见。
*   **模块合并**:
    *   废弃独立的“审核”与“管理”页面。
    *   统一为 **“信息管理中心”**，通过 Tab 切换状态：`待审核` | `已发布` | `已驳回` | `回收站`。
*   **管理员权限**:
    *   **先隐后审**: 对于 *已发布* (APPROVED) 但存在争议的内容，管理员可将其状态改为 `HIDDEN` (隐藏/下架)，前台不可见，待进一步核查。
    *   **逻辑删除**: 管理员删除内容仅做逻辑标记，数据保留在数据库。

### 2.5 数据与日志
*   **操作日志 (operation_log)**:
    *   记录关键动作：`LOGIN`, `DELETE`, `UPDATE_PASSWORD`, `AUDIT`.
    *   字段：操作人ID、操作时间、IP、操作类型、操作详情 (JSON)。
*   **分页策略**:
    *   后端 Mybatis-Plus 分页插件配置，确保全量列表均为后端分页。

## 3. 数据库变更计划 (Draft)

```sql
-- 1. 用户表新增
ALTER TABLE user ADD COLUMN account_name VARCHAR(50) UNIQUE COMMENT '账户名';
ALTER TABLE user ADD COLUMN student_id VARCHAR(50) UNIQUE COMMENT '学号';
ALTER TABLE user ADD COLUMN college VARCHAR(100) COMMENT '学院';
ALTER TABLE user ADD COLUMN status TINYINT DEFAULT 0 COMMENT '状态 0:正常 1:封禁';
ALTER TABLE user ADD COLUMN is_deleted TINYINT DEFAULT 0 COMMENT '逻辑删除';

-- 2. 失物表新增
ALTER TABLE lost_info ADD COLUMN lost_time DATETIME COMMENT '丢失时间';
ALTER TABLE lost_info ADD COLUMN contact_info VARCHAR(100) COMMENT '联系方式';
ALTER TABLE lost_info ADD COLUMN is_deleted TINYINT DEFAULT 0 COMMENT '逻辑删除';

-- 3. 招领表新增
ALTER TABLE find_info ADD COLUMN found_time DATETIME COMMENT '捡到时间';
ALTER TABLE find_info ADD COLUMN contact_info VARCHAR(100) COMMENT '联系方式';
ALTER TABLE find_info ADD COLUMN is_deleted TINYINT DEFAULT 0 COMMENT '逻辑删除';

-- 4. 新增日志表
CREATE TABLE operation_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT COMMENT '操作人',
    action_type VARCHAR(50) COMMENT '操作类型',
    content TEXT COMMENT '详情',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 5. 新增白名单表
CREATE TABLE student_whitelist (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    student_id VARCHAR(50) UNIQUE COMMENT '学号',
    name VARCHAR(50) COMMENT '姓名',
    college VARCHAR(100) COMMENT '学院'
);
```
