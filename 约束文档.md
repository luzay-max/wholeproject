# 开发约束文档 (Development Constraints)

为保证代码质量与风格统一，所有开发人员需严格遵守以下约束。

## 1. 命名规范

### 1.1 Java 后端
*   **类名**: UpperCamelCase (如 `UserController`).
*   **方法/变量**: lowerCamelCase (如 `getUserById`).
*   **常量**: UPPER_CASE_WITH_UNDERSCORES (如 `MAX_RETRY_COUNT`).
*   **包名**: 全小写 (如 `com.lzy.lostandfound.controller`).
*   **接口实现**: 接口名以 `I` 开头 (如 `IUserService`), 实现类以 `Impl` 结尾 (如 `UserServiceImpl`)。

### 1.2 Vue 前端
*   **组件文件**: UpperCamelCase (如 `LostList.vue`).
*   **文件夹**: lowerCamelCase (如 `components/common`).
*   **CSS 类名**: kebab-case (如 `user-profile-card`).

### 1.3 数据库
*   **表名**: snake_case (如 `lost_info`).
*   **字段名**: snake_case (如 `create_time`).
*   **布尔字段**: 以 `is_` 开头 (如 `is_deleted`).

## 2. 编码规范

### 2.1 后端开发
*   **Controller**: 仅做参数接收与简单校验，业务逻辑必须下沉至 Service 层。
*   **Service**: 事务控制 (`@Transactional`) 加在 Service 方法上。
*   **注释**: 
    *   所有 Controller 接口需加 Swagger/Javadoc 注释。
    *   复杂业务逻辑需加行级注释。
*   **魔法值**: 禁止在代码中直接使用魔法值 (Magic Number/String)，需定义常量或枚举。
*   **日志**: 使用 `log.info`/`log.error`，禁止使用 `System.out.println`。
*   **接口唯一性**: 同一控制器不得定义重复的路径与方法映射（如重复的 `@DeleteMapping("/{id}")`）。
*   **实体同步**: 在 Controller/DTO 使用新字段前，实体需同步新增并提供 `getter/setter`，保持 Mapper/Service 一致。
*   **状态规范**: 所有业务状态值统一为大写（`PENDING/APPROVED/REJECTED/SOLVED`），入库前做规范化。
*   **时间校验**: 业务时间字段（`lostTime/foundTime`）不得晚于当前时间，统一使用 `Asia/Shanghai` 时区进行校验。
*   **逻辑删除**: 删除接口统一采用逻辑删除（`@TableLogic`），普通查询默认排除已删除数据。
*   **构建约束**: `maven-compiler-plugin` 固定 `release=17`；Lombok 使用 `1.18.34` 并作为注解处理器，避免 JDK/插件版本不匹配。
*   **配置统一**: 后端配置文件统一为 `application.yaml`，禁止与 `application.yml` 共存。
*   **令牌策略**: Access Token 存入 Redis 并设置过期；Remember Me 使用前缀 `refresh:` 的 Refresh Token，过期 7 天。
*   **AOP 使用**: 使用切面需引入 `spring-boot-starter-aop`；切面不得吞异常（至少记录 error 日志），避免静默失败。
*   **操作日志**: 日志实体与数据库表结构必须一致；日志内容以 JSON 存储，自动过滤敏感字段（如 `password/captcha`）；允许异步写库但失败需记录错误。

### 2.2 前端开发
*   **API 调用**: 所有后端请求必须封装在 `src/api` 目录下，禁止在 Vue 组件中直接调用 `axios`。
*   **样式**: 避免行内样式，尽量使用 Class。
*   **组件**: 通用功能必须封装为组件。

## 3. Git 提交规范 (Conventional Commits)
格式: `<type>(<scope>): <subject>`

*   `feat`: 新功能
*   `fix`: 修复 Bug
*   `docs`: 文档变更
*   `style`: 代码格式调整 (不影响逻辑)
*   `refactor`: 重构 (无新功能/Bug修复)
*   `chore`: 构建/工具链变动

**示例**: `feat(user): 增加用户登录验证码校验`

## 4. 安全约束
*   **SQL 注入**: 严禁拼接 SQL，必须使用 MyBatis `#{}` 占位符。
*   **敏感数据**: 密码必须加盐 Hash 存储（推荐 BCrypt，或 MD5+Salt），禁止明文或无盐 MD5。
*   **越权访问**: 所有非公开接口必须校验当前用户身份及权限。
